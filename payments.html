Payments

Payment Processing

Quickpay provides a comprehensive payment processing system that supports multiple payment methods and currencies.

Payment Methods

  • Credit/Debit Cards (Visa, Mastercard, etc.)
  • MobilePay
  • Apple Pay
  • Google Pay
  • Bank Transfers

Payment Flow

  1. Create a payment
  2. Authorize the payment
  3. Capture the payment
  4. Handle refunds if needed

API Endpoints

Endpoint Method Description
/payments POST Create a new payment
/payments/{id} GET Get payment details
/payments/{id}/authorize POST Authorize a payment
/payments/{id}/capture POST Capture an authorized payment
/payments/{id}/refund POST Refund a payment

Payment States

  • Initial - Payment created but not processed
  • Pending - Payment is being processed
  • Authorized - Payment is authorized but not captured
  • Captured - Payment is completed
  • Refunded - Payment has been refunded
  • Failed - Payment failed

Example Payment Creation

POST /payments
{
    "currency": "DKK",
    "order_id": "order_123",
    "amount": 10000,
    "payment_methods": ["creditcard", "mobilepay"],
    "description": "Order #123",
    "variables": {
        "customer_id": "cust_123"
    }
}
