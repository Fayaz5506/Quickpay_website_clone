Quickpay Form

Overview

Quickpay Form is a customizable payment form that you can embed directly into your website. It provides more flexibility than Quickpay Link while maintaining security and compliance.

Features

  • Embeddable payment form
  • Full customization options
  • Multiple payment methods
  • Responsive design
  • Built-in validation

Integration Steps

  1. Include the Quickpay Form JavaScript library
  2. Create a payment in your backend
  3. Initialize the form with your payment details
  4. Handle the payment response

Basic Implementation

<!-- Include Quickpay Form library -->
<script src="https://payment.quickpay.net/form/form.js"></script>

<!-- Create a container for the form -->
<div id="quickpay-form"></div>

<script>
// Initialize the form
QuickpayForm.init({
    payment_id: 'payment_id',
    merchant_id: 'your_merchant_id',
    amount: 10000,
    currency: 'DKK',
    payment_methods: ['creditcard', 'mobilepay'],
    callback_url: 'https://your-domain.com/callback'
});
</script>

Form Configuration Options

Option Description
payment_id ID of the created payment
merchant_id Your Quickpay merchant ID
amount Payment amount in smallest currency unit
currency Payment currency (e.g., DKK, EUR)
payment_methods Array of enabled payment methods
callback_url URL to receive payment notifications

Styling and Customization

// Custom styling example
QuickpayForm.init({
    // ... other options ...
    styling: {
        primary_color: '#0066cc',
        secondary_color: '#f5f5f5',
        font_family: 'Arial, sans-serif',
        button_radius: '4px',
        input_radius: '4px'
    }
});

Event Handling

QuickpayForm.on('payment_success', function(response) {
    // Handle successful payment
    console.log('Payment successful:', response);
});

QuickpayForm.on('payment_error', function(error) {
    // Handle payment error
    console.error('Payment failed:', error);
});

Security Considerations

  • Always use HTTPS
  • Validate payment data server-side
  • Implement proper error handling
  • Use test mode during development
