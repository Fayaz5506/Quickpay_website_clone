Callback

Callback System

The callback system allows you to receive real-time notifications about payment status changes and other important events.

How Callbacks Work

When a payment status changes or an important event occurs, Quickpay will send a POST request to your callback URL with relevant information about the event.

Setting Up Callbacks

To set up callbacks, you need to:

  1. Configure your callback URL in the Quickpay dashboard
  2. Implement an endpoint to receive and process callbacks
  3. Verify the callback signature for security

Callback Data Structure

{
    "id": "payment_id",
    "order_id": "your_order_id",
    "accepted": true,
    "test_mode": false,
    "branding_id": null,
    "variables": {},
    "acquirer": "clearhaus",

    "metadata": {},
    "created_at": "2024-03-20T12:00:00Z",
    "updated_at": "2024-03-20T12:00:00Z",
    "retented_at": null,
    "status": {
        "code": "20000",
        "message": "Approved"
    }
}

Security Considerations

  • Always verify the callback signature
  • Use HTTPS for your callback endpoint
  • Implement proper error handling
  • Store callback logs for debugging

Best Practices

  • Respond quickly to callbacks (within 5 seconds)
  • Implement idempotency to handle duplicate callbacks
  • Monitor callback failures
  • Keep your callback URL up to date

Testing Callbacks

You can test your callback implementation using our test environment. Test callbacks will be marked with test_mode: true.
